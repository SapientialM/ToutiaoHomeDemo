## 1. 项目背景与目标

### 1.1 背景
字节跳动客户端工程训练营课题要求开发一个头条首页列表页 Demo，通过该课题考察参赛者的 Android 开发技能、工程架构理解、技术文档撰写能力以及对新闻应用业务的理解。

### 1.2 核心目标
- 还原今日头条首页核心信息流体验
- 运用 Kotlin、Jetpack Compose、协程、Room 等现代 Android 技术栈
- 体现良好的代码规范、架构设计（MVI / MVVM）与工程素养
- 为后续答辩与绿色面试通道提供技术展示载体

## 3. 非功能需求

### 3.1 性能需求
- 列表滑动帧率 ≥ 55fps（Compose 重组优化）
- 首屏加载时间 ≤ 1.5s
- 图片加载使用内存缓存，缓存上限 50MB

### 3.2 兼容性需求
- 最低支持 Android API 26（Android 8.0）
- 适配不同屏幕尺寸（手机竖屏为主）

### 3.3 研发素养需求
- 代码规范：遵循 Kotlin 编码规范，使用 ktlint 检查
- Git 规范：Commit 信息清晰，分支管理合理
- 文档完整：README、技术文档、进度文档齐全

## 4. 业务理解

### 4.1 头条首页核心业务流程
```
用户打开 App → 加载推荐频道 → 展示信息流卡片 → 用户滑动浏览
    ↓                              ↓
触发下拉刷新 ← 滑动到底部加载更多 ← 点击卡片进入详情
```

### 4.2 卡片类型分析（基于产品截图）

| 卡片类型 | 特征 | 业务含义 |
|----------|------|----------|
| 纯文本置顶卡 | 标题 + "置顶"标签 + 来源 + 评论数 | 重要新闻，固定顶部 |
| 左文右图卡 | 左侧标题/来源/评论/时间 + 右侧缩略图 | 常规图文资讯 |
| 大图卡 | 标题 + 底部大图 | 视觉冲击力强的内容 |
| 视频卡 | 标题 + 封面图 + 播放按钮 + 时长 | 视频内容分发 |

### 4.3 关键业务指标（工程视角）
- **曝光率**：卡片进入可视区域即算曝光（LazyColumn 可见性检测）
- **点击率**：用户点击卡片进入详情页
- **刷新率**：用户主动下拉刷新的频次
- **加载成功率**：网络请求成功 / 总请求数

## 5. 需求边界

### 5.1 明确在范围内
- ✅ 首页信息流列表展示
- ✅ 多种卡片类型渲染
- ✅ 下拉刷新、加载更多
- ✅ 本地缓存与离线展示
- ✅ Tab 切换（至少 2 个频道）
- ⬜ Mock 数据源独立组件化（Repository 层无感知，支持错误模拟与延迟模拟）

### 5.2 明确不在范围内
- ❌ 用户登录/注册系统
- ❌ 新闻详情页完整实现（可预留跳转入口）
- ❌ 视频播放功能（仅展示封面与播放按钮 UI）
- ❌ 评论系统、点赞、分享等社交功能
- ❌ 推送通知
- ❌ 商城、个人中心等独立页面（仅底部导航 UI）

## 6. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Mock API 不稳定 | 数据加载受阻 | 准备本地 JSON 兜底数据 |
| Compose 性能问题 | 列表卡顿 | 提前学习重组优化策略，使用 key 和 remember |
| 时间不足 | 功能无法全部完成 | 按 P0/P1/P2 优先级取舍，保证基础功能质量 |
| 答辩追问深度问题 | 答不上来 | 提前准备高频问题答案，理解而非死记 |
