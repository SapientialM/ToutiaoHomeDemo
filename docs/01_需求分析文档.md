## 1. 项目背景与目标

### 1.1 背景
字节跳动客户端工程训练营课题要求开发一个高度还原今日头条首页的 Demo。通过该课题考察参赛者的 Android 开发技能、工程架构理解、技术文档撰写能力，以及对新闻客户端核心业务场景（信息流分发、多形态卡片、频道切换、置顶策略）的深入理解。

### 1.2 核心目标
- **视觉还原**：基于产品截图，1:1 还原顶部天气栏、搜索栏、频道 Tab、信息流卡片、底部导航等关键 UI。
- **技术栈落地**：运用 Kotlin、Jetpack Compose、Paging3、Room、Hilt、协程等现代 Android 技术栈。
- **业务逻辑完整**：支持频道切换、下拉刷新、加载更多、置顶排序、多卡片类型渲染、本地缓存与离线展示。
- **工程素养体现**：MVI + Clean Architecture 架构、单向数据流、完善的日志与调试能力、代码规范与文档完整。

### 2.3 信息流卡片（核心）

根据产品截图，信息流至少包含 **4 种卡片类型**：

#### ① 纯文本置顶卡（TextTop）
- **布局**：仅文字，无图片。
- **内容**：标题 + 红色"置顶"标签 + 来源 + 评论数。
- **特征**：多条置顶新闻可在列表顶部连续出现。
- **业务含义**：权威来源（如新华社、人民日报、央视新闻等）的重要新闻，需在每日同日期新闻中优先展示。

#### ② 左文右图卡（LeftTextRightImage）
- **布局**：左侧文字区 + 右侧缩略图。
- **内容**：标题、来源、评论数、发布时间；右侧为 120×80dp 左右的封面图。
- **特征**：右下角显示"×"关闭按钮（预留交互，可点击移除单条）。

#### ③ 视频卡（Video）
- **布局**：左侧文字区 + 右侧封面大图。
- **内容**：标题、来源、评论数、发布时间；右侧封面图中央叠加播放按钮，右下角显示视频时长（如 `02:54`）。
- **特征**：右下角显示"×"关闭按钮；点击封面或播放按钮预留视频播放入口。

#### ④ 底部大图卡（LargeImage / 多图）
- **布局**：标题 + 底部大图（单张或多张横图拼接）。
- **内容**：标题独占一行，下方为 16:9 或更宽幅的图片；多张横图时可横向并排展示。
- **特征**：视觉冲击力强，用于热点事件、娱乐综艺等内容。

| 编号 | 功能模块 | 需求描述 | 验收标准 |
|------|----------|----------|----------|
| F-09 | 卡片类型识别 | 根据 DTO 的 `type` 字段分发到对应卡片组件 | 4 种卡片渲染正确，无类型错乱 |
| F-10 | 置顶标签渲染 | `isTop=true` 时，卡片标题旁显示红色"置顶"角标 | 仅 TextTop 默认带置顶标签，其他类型有 `isTop` 字段支持 |
| F-11 | 图片加载 | 使用 Coil 加载网络图片，支持内存缓存与尺寸裁剪 | 滑动流畅，无白块、无内存泄漏 |
| F-12 | 关闭按钮（预留） | 左文右图卡与视频卡右下角显示"×" | 点击可上报事件（暂不做真实移除动画） |

## 3. 非功能需求

### 3.1 性能需求
- 列表滑动帧率 ≥ 55fps（Compose 重组优化、`@Immutable`、`key` 正确使用）。
- 首屏加载时间 ≤ 1.5s（Room 缓存优先）。
- 图片内存缓存上限 50MB，使用 Coil `ImageLoader` 自定义配置。

### 3.2 兼容性需求
- 最低支持 Android API 26（Android 8.0）。
- 目标设备为手机竖屏，适配主流屏幕密度（xhdpi、xxhdpi）。

### 3.3 研发素养需求
- 代码规范：遵循 Kotlin 官方编码规范，禁止 `!!`，函数体不超过 30 行。
- 架构约束：MVI 单向数据流（UiState / UiEvent / ViewModel），Presentation 层不直接引用 Data 层。
- Git 规范：Commit 信息中文，格式 `<类型>: <简述>`。
- 文档完整：README、技术设计文档、本需求文档、进度文档齐全。

## 4. 业务理解

### 4.1 头条首页核心业务流程

```
用户打开 App → 加载推荐频道 → 展示信息流卡片
      ↓                              ↓
  下拉刷新 ← 滑动到底部加载更多 ← 点击卡片进入详情（预留）
      ↓
切换频道 Tab → 新频道数据重置 → 滚动条回顶
```

### 4.2 信息流卡片类型对照（截图 → 代码映射）

| 截图中的视觉形态 | 对应代码中的卡片类型 | 核心特征 |
|------------------|----------------------|----------|
| 多条纯文字 + 红色"置顶"标签 | `FeedCard.TextTop` | 无图，来源权威，同日期置顶优先 |
| 左侧标题文字 + 右侧小图 | `FeedCard.LeftTextRightImage` | 常规图文，信息密度高 |
| 左侧标题文字 + 右侧视频封面（带播放按钮+时长） | `FeedCard.Video` | 视频分发，封面吸睛 |
| 标题 + 底部大图/多图 | `FeedCard.LargeImage` | 视觉重内容，适合热点 |

### 4.3 置顶业务规则（关键）

1. **置顶判定**：由后端（或 MockDataSource）根据新闻来源判定。权威来源（新华社、人民日报、央视新闻、教育部、国防部等）标记为 `isTop = true`。
2. **置顶展示范围**：置顶新闻在同一天的频道内置顶；跨天时，旧置顶不再置顶（或按产品策略降级）。
3. **排序公式**：数据库查询必须遵循 `ORDER BY 日期 DESC, 置顶 DESC, 时间 DESC`，确保同一天内置顶新闻始终排在非置顶新闻之前。

## 5. 需求边界

### 5.1 明确在范围内 ✅
- 首页信息流列表展示（4 种卡片类型）。
- 顶部天气栏、搜索栏、频道 Tab、底部导航的 UI 还原。
- 频道切换与数据隔离。
- 下拉刷新、上拉加载更多。
- Room 本地缓存与离线展示。
- 置顶新闻排序与标签渲染。
- Mock 数据源独立组件化（支持延迟/错误模拟）。

### 5.2 明确不在范围内 ❌
- 用户登录/注册系统。
- 新闻详情页完整实现（仅预留点击事件）。
- 视频播放功能（仅封面 + 播放按钮 UI + 时长标签）。
- 评论系统、点赞、分享、收藏等社交功能。
- 推送通知与消息中心。
- "视频/任务/我的"等独立页面的完整业务逻辑（底部导航纯视觉占位）。
- AI 回答、发布、关闭按钮的真实业务逻辑（预留 UI 与点击事件）。

## 6. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 截图与真实产品存在迭代差异 | 需求理解偏差 | 以当前提供的两张截图为准，不追最新版 App 变化 |
| Compose 重组导致列表卡顿 | 用户体验差 | 使用 `@Immutable`、`LazyColumn` 指定 `key`、避免 Composable 中直接计算 |
| Paging3 + RemoteMediator 状态机复杂 | 加载异常、死循环 | 严格遵循 Paging3 协议，`initialize()` 不做数据清理，`load(REFRESH)` 中统一清理 |
| 置顶排序与分页逻辑冲突 | 同一页内置顶插队导致页码错乱 | 排序以"日期"为第一优先级，分页以"页码"为物理偏移，Mediator 页码持久化到 `remote_keys` |
| 时间不足 | 功能无法全部完成 | 按本文档优先级取舍：卡片渲染 > 频道切换 > 置顶排序 > 搜索/底部导航细节 |

### 7.2 接口定义

```
GET /api/news/feed
参数：
  - channel: String (频道 ID，如 "recommend", "hot", "video", "society", "tech")
  - page: Int (页码，0 起始)
  - size: Int (每页条数，默认 20)

响应：
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": "string",
        "type": "text_top | left_text_right_image | large_image | video",
        "title": "string",
        "source": "string",
        "commentCount": 0,
        "imageUrl": "string",
        "videoUrl": "string",
        "duration": "string",
        "publishTime": "string",
        "isTop": false,
        "createdAt": 1716888888000
      }
    ],
    "hasMore": true
  }
}
```

**字段说明**：
- `type`：决定卡片渲染类型。
- `isTop`：置顶标记，影响排序与同日期内的展示优先级。
- `createdAt`：新闻原始发布时间戳（毫秒），用于 Room 排序，**禁止使用插入时间**。
- `hasMore`：是否还有更多页，用于控制分页结束。
