**今日心得/技术收获：**
- Sealed class 非常适合表达异构列表的卡片类型，配合 Compose 的 when 表达式实现类型安全的多卡片渲染

**今日心得/技术收获：**

- Compose Preview 应当覆盖所有 UiState 分支，设计阶段就能在 IDE 内验证 UI 效果
- Hilt 的 `@Binds`（接口-实现绑定）与 `@Provides`（工厂方法）各自适用的场景不同：Repository 用 Binds，API/Database 用 Provides

### 5月24日（周日）

**今日完成：**
- [x] 接入 Paging3 + RemoteMediator：替换原有手动分页逻辑
  - 新建 `NewsRemoteMediator`：处理 `REFRESH`（清空旧数据）和 `APPEND`（加载下一页）
  - `FeedDao` 新增 `getFeedPagingSource(channel)` 返回 `PagingSource<Int, FeedItemEntity>`
  - `NewsRepository` 新增 `getFeedPagingData(channel): Flow<PagingData<FeedCard>>`
  - `HomeViewModel` 改用 `Pager.flow.cachedIn(viewModelScope)`，移除 `currentPage` 手动维护
  - `HomeScreen` 改用 `LazyPagingItems` + `collectAsLazyPagingItems()`，由 `loadState` 驱动 Loading/Error/Empty/列表状态
- [x] Room 缓存优先离线展示（Paging3 自动实现）
  - RemoteMediator 网络请求成功后自动写入 Room
  - PagingSource 从 Room 读取，`Flow` 自动触发 UI 更新
  - 断网时 Room 缓存自动展示（无需额外 ViewModel 逻辑）
- [x] 搜索栏交互完整实现
  - `HomeUiEvent` 新增 `OnSearchClicked` / `OnSearchQueryChanged` / `OnSearchSubmit` / `OnSearchDismiss`
  - `HomeUiState.Success` 新增 `searchQuery` / `isSearching`
  - 顶部栏支持搜索占位 → 输入框状态切换
  - `NewsRepository.searchNews(query)` Mock 返回 3 条搜索结果
- [x] 性能优化基础
  - `FeedCard` 密封类添加 `@Immutable`，减少 Compose 重组范围
  - Room 数据库开启 WAL 模式（`JournalMode.WRITE_AHEAD_LOGGING`）
  - `feed_items` 表对 `channel` 字段建立索引
  - `ToutiaoApplication` 实现 `ImageLoaderFactory`，配置 Coil 内存缓存（`maxSizePercent(0.25)`）
- [x] 编译验证通过：`./gradlew :app:compileDebugKotlin` BUILD SUCCESSFUL

**明日计划：**
- 真机运行验证 Tab 切换、下拉刷新、加载更多、搜索交互
- 补充 Compose 重组分析

**阻塞问题：**
- 无

**今日心得/技术收获：**
- Paging3 + RemoteMediator 是官方推荐的网络+本地混合分页方案：`RemoteMediator` 负责网络请求和写入 Room，`PagingSource` 只从 Room 读。Room 的 Flow 特性会自动通知 UI 更新，天然实现离线展示。
- `LazyPagingItems.loadState.refresh` / `.append` 可以完整覆盖首次加载、下拉刷新、加载更多、错误、空态等场景，但和传统的 `HomeUiState` 密封类有一定重叠。实际项目中需要权衡是否保留 `HomeUiState` 给非列表状态（如搜索态）。
- Compose Preview 中可以使用 `flowOf(PagingData.from(mockItems)).collectAsLazyPagingItems()` 来模拟 `LazyPagingItems`，方便在 IDE 中预览 Paging3 列表 UI。
