## 3. 日报

### 5月22日（周五）

**今日完成：**
- [x] 项目骨架搭建：创建 MVI + Clean Architecture 完整包结构
- [x] 构建配置：添加 Hilt、Room、Retrofit、Coil、Paging3、Kotlinx Serialization 等依赖
- [x] 领域层：FeedCard 密封类（4 种卡片类型）、NewsRepository 接口
- [x] 数据层：NewsApi（Retrofit）、DTO（Kotlinx Serialization）、Room Entity/DAO/Database、Repository 实现、数据映射器
- [x] 表示层：HomeScreen（含顶部 Tab + 底部导航 + LazyColumn 多卡片渲染）、HomeViewModel（MVI）、UiState/UiEvent 密封类
- [x] 4 种卡片组件：TextTopCard、LeftTextRightImageCard、LargeImageCard、VideoCard
- [x] DI 模块：Hilt NetworkModule + DatabaseModule
- [x] 主题：头条红白配色方案（Color/Type/Theme）
- [x] 入口：HiltApplication + MainActivity（@AndroidEntryPoint）
- [x] AndroidManifest：添加 INTERNET 权限，注册 Hilt Application

**明日计划：**
- 确认项目可以编译通过
- 准备 Mock 数据或本地 JSON 兜底数据
- 实现首页首次加载数据流程连通

**阻塞问题：**
- 无

**今日心得/技术收获：**
- Sealed class 非常适合表达异构列表的卡片类型，配合 Compose 的 when 表达式实现类型安全的多卡片渲染
- Hilt + KSP 的编译期依赖注入比传统 kapt 更快，Room 2.7+ 也全面支持 KSP
- MVI 的 UiState 密封类能精确描述 Loading/Success(含 isRefreshing/isLoadingMore)/Error/Empty 四种状态，避免传统 MVVM 中多个 LiveData 的状态不一致问题

### 5月23日（周六）

**今日完成：**
- [x] 真机调试：修复 Mock 数据流不连通问题，App 成功渲染 4 种卡片
- [x] 创建 MockInterceptor：OkHttp 拦截器，拦截 API 请求返回预置 JSON（支持分页、channel 参数）
- [x] 数据链路打通：HomeViewModel → Repository → Retrofit → MockInterceptor → DTO → Entity → FeedCard → UI
- [x] 添加 HomeScreen 多状态 Preview（Success/Loading/Error/Empty/Refreshing + 8 条 mock 数据）
- [x] 全链路 Timber 日志：ViewModel → Repository → MockInterceptor，方便追踪数据流
- [x] JSON 解析配置优化：ignoreUnknownKeys / isLenient / coerceInputValues
- [x] 创建 AGENT.md：AI Agent 协作协议与项目开发约束
- [x] 创建 README.md：含项目效果图、技术栈、架构图、构建指南
- [x] Git 初始提交：9 个 commit，完整版本历史（`c1531e2` → `6e0df20`）
- [x] 文档更新：01/02/03/04 同步至 v1.1，补充实际项目结构与调试说明

**阻塞问题：**
- ~~Retrofit baseUrl 占位导致网络超时~~ → 已解决：MockInterceptor 零延迟返回 Mock JSON
- ~~Room 依赖未注入导致 Hilt 编译报错~~ → 已解决：RepositoryModule @Binds 绑定接口

**今日心得/技术收获：**
- OkHttp Interceptor 是实现 Mock API 的最佳方案：零侵入（不修改 Repository/ViewModel 代码），上线时移除一行 `.addInterceptor()` 即可切换到真实 API
- Compose Preview 应当覆盖所有 UiState 分支，设计阶段就能在 IDE 内验证 UI 效果
- Hilt 的 `@Binds`（接口-实现绑定）与 `@Provides`（工厂方法）各自适用的场景不同：Repository 用 Binds，API/Database 用 Provides
